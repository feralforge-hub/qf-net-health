# Public Publication Checklist

Use this checklist before committing files to this public repository.

## Content Safety

- No passwords, tokens, API keys, or credentials
- No private keys or certificates
- No real `.env` files
- No customer or employer data
- No personally sensitive information

## Infrastructure Safety

- No real internal IP addresses
- No real hostnames
- No router or firewall exports with private values
- No screenshots exposing dashboards or internal systems

## Sanitization Standards

- Replace values with placeholders
- Use documentation-only examples
- Remove hidden metadata from files when practical
- Re-read content as if it were visible to the public internet

## Final Gate

If unsure whether something should be public, do not publish until reviewed.
